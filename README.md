# ENTRANCE-EXAM-MARKS-AND-RANK-ALLOCATION
This project is about an entrance exam system where students’ data is processed, ranks are allocated, and operations like searching, sorting, and queue/stack management are demonstrated. It’s a project where DSA (Data Structures and Algorithms) concepts are applied to a real world scenario of exam rank allocation.

 Project Structure
FileOperations.java:  
  Handles reading/writing student data to files.

HashGateKeep.java:
  Implements hashing logic for quick student lookup and duplicate prevention.

InpData.java:
  Manages input data collection (student details, marks, etc.).

KLUentryExamm.java:  
  Main driver class simulating the exam process and rank allocation.

Nodey.java: 
  Defines a node structure for linked list implementations.

QueueOperations.java:  
  Demonstrates queue based operations (Allotment of ranks and scholarship percentage using priority queue ).

QuNode.java: 
  Node structure specific to queue implementation.

Searching.java:  
  Implements searching algorithms (binary) to find students by ID.

Sorting.java:  
  Implements sorting algorithms (merge) to rank students by Hall Ticekt for better data management.

StackOperations.java:
  Demonstrates stack-based operations (Verifying Hall Ticket Pattern of '25101').

student (Text Document):  
  file for storing student records.

- Student registration and data input
- Hall Ticekt Sorting using sorting algorithms
- Search functionality for student records
- Queue simulation for rank simulation
- Stack simulation for hall ticket processing
